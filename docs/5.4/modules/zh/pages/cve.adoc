== OpenID Connect 易受中间人攻击 (MITM) 的影响。

* **CVE ID:** https://nvd.nist.gov/vuln-metrics/cvss/v3-calculator?vector=AV:N/AC:L/PR:N/UI:R/S:U/C:H/I:H/A:H&version=3.1[CVE-2025-66001 (AV:N/AC:L/PR:N/UI:R/S:U/C:H/I:H/A:H)]
* **CVSS 分数:**8.8

=== 受影响的版本

* 所有早于 `5.3.0` 的版本
* 版本 `5.3.0` 到 `5.4.7`

*修复版本:* `5.4.8`

=== 影响

NeuVector 支持使用 OpenID Connect 进行身份验证。默认情况下，不强制执行 TLS 验证，该验证用于确保远程服务器的真实性和完整性。这可能会使系统暴露于中间人（MITM）攻击。

从版本 `5.4.0` 开始，NeuVector 支持 TLS 验证，适用于：

* 注册表连接
* 身份验证服务器连接（SAML、LDAP 和 OIDC）
* Webhook 连接

默认情况下，TLS 验证保持禁用。该设置在 NeuVector UI 的 *设置 → 配置* 下可用。

在修补版本中，新的 NeuVector 部署默认启用 TLS 验证。滚动升级不会修改现有配置，以防止服务中断。

[NOTE]
====
当启用 TLS 验证时，适用于：

* 注册服务器
* 身份验证服务器（SAML、LDAP 和 OIDC）
* Webhook 服务器
====

=== 增补程序

修补版本包括 `v5.4.8` 及以后的版本。

=== 解决方案

手动启用 TLS 验证：

. 打开 NeuVector 用户界面。
. 导航到 *设置 → 配置*。
. 在 **TLS 自签名证书配置** 中，选择 *启用 TLS 验证*。
. （可选）上传或粘贴 TLS 自签名证书。

== NeuVector 正在其二进制文件中嵌入加密材料。

* **CVE ID：**CVE-2025-54471
* **CVSS 分数:**6.5 – https://nvd.nist.gov/vuln-metrics/cvss/v3-calculator?vector=AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:N/A:N&version=3.1[AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:N/A:N]

=== 受影响的版本

* 所有早于 `5.3.0` 的版本
* 版本 `5.3.0` 到 `5.4.6`

*修复版本:* `5.4.7`

=== 影响

NeuVector 之前使用嵌入源代码中的硬编码加密密钥。在编译过程中，此值被替换为用于加密敏感配置字段的静态密钥。

在修补版本中，NeuVector 使用 Kubernetes Secret `neuvector-store-secret`（在 `neuvector` 名称空间中）生成安全的动态加密密钥。这消除了对静态密钥的依赖，并通过将密钥存储在 Kubernetes 管理的 Secret 中来提高安全性。

在滚动升级或从持久存储恢复时，NeuVector 控制器检查加密的配置字段。如果字段使用默认固定密钥加密，则会使用新的动态密钥进行解密和重新加密。

如果控制器没有访问 Kubernetes Secret 的 RBAC 权限，则会记录：

`Required Kubernetes RBAC for secrets are not found`

并退出。

设备加密密钥每三个月轮换一次。有关详细信息，请参见：
xref:certrotate.adoc#_rotating_sensitive_field_in_configuration[在配置中轮换敏感字段]。

=== 增补程序

修补版本包括 `v5.4.7` 及以后的版本。

=== 解决方案

没有可用的解决方法。请尽快升级到修补版本。

== 遥测发送器易受中间人攻击和拒绝服务攻击的影响。

* **CVE ID：**CVE-2025-54470
* **CVSS 分数:**8.6 – https://nvd.nist.gov/vuln-metrics/cvss/v3-calculator?vector=AV:N/AC:L/PR:N/UI:N/S:U/C:L/I:L/A:H&version=3.1[AV:N/AC:L/PR:N/UI:N/S:U/C:L/I:L/A:H]

=== 受影响的版本

* 所有早于 `5.3.0` 的版本
* 版本 `5.3.0` 到 `5.4.6`

*修复版本：* `5.4.7`, `5.3.5`

=== 影响

此漏洞仅在启用 **报告匿名集群数据** 时影响 NeuVector 部署。启用时，NeuVector 将匿名遥测数据发送到：

`https://upgrades.neuvector-upgrade-responder.livestock.rancher.io`

在受影响的版本中，未强制执行 TLS 证书验证，使遥测通信易受中间人攻击。攻击者可以拦截或修改传输的数据。

NeuVector 还在内存中加载了遥测服务器响应，没有大小限制，暴露系统于拒绝服务（DoS）风险。

修补版本包括：

* 对遥测服务器的 TLS 证书链和主机名进行验证。
* 响应大小限制为 256 字节，以减轻内存耗尽。

这些改进默认启用，无需用户操作。

=== 增补程序

修补版本包括 `v5.4.7` 及以后的版本。

=== 解决方案

如果您无法更新，请禁用**报告匿名集群数据**：

*设置 → 配置 → 报告匿名集群数据*

禁用此设置将阻止 NeuVector 发送遥测数据，从而减少对该漏洞的暴露。

*建议：*请尽快升级到修补版本。

== NeuVector Manager 容器日志中的敏感信息泄露

*CVEs:*CVE-2025-46808 +
*CVSS评分:*6.8 — https://nvd.nist.gov/vuln-metrics/cvss/v3-calculator?vector=AV:N/AC:L/PR:L/UI:R/S:C/C:H/I:N/A:N&version=3.1[CVSS v3.1向量] +
*CWE:* https://cwe.mitre.org/data/definitions/532[CWE-532：将敏感信息插入日志文件]

=== 受影响的版本

* 所有早于 `5.0.0` 的版本
* 版本 `5.0.0` 到 `5.4.4`

*修复版本:* `5.4.5`

=== 影响

NeuVector版本高达并包括`5.4.4`的漏洞可能会在NeuVector Manager容器日志中泄露敏感信息。以下字段可能出现在日志中：

[cols="1,3,2,2,1", options="header"]
|===
| 字段 | 字段描述 | 出现的地方 | 重现 | 环境

| `X-R-Sess`
| 用于单点登录的Rancher会话令牌
| 请求头
| 通过Rancher UI登录并访问NeuVector SSO
| 带有NeuVector SSO的Rancher

| `personal_access_token`
| GitHub或Azure DevOps令牌
| 请求体
| 在 *配置 > 设置* 下提交远程储存库配置
| NeuVector

| `token1.token`
| NeuVector 用户会话令牌
| 响应体
| 通过 NeuVector API 发送 GET 请求：`https://<neuvector-ui-url>/user?name=<username>`
| NeuVector

| `rekor_public_key`、`root_cert`、`sct_public_key`
| Rekor 公钥、根证书、签名证书时间戳 (SCT) 公钥在私有信任根中
| 请求体
| 从 Sigstore 页面创建或更新私有信任根
| NeuVector

| `public_key`
| 验证者的公钥
| 请求体
| 在 Sigstore 页面创建或更新验证者
| NeuVector
|===

[NOTE]
====
与 Rancher Manager *和* 远程储存库配置禁用的单点登录集成的 NeuVector 安装不受影响。
====

在补丁版本中，`X-R-Sess` 部分被屏蔽。其他敏感字段 (`personal_access_token`, `token`, `rekor_public_key`, `root_cert`, `sct_public_key`, `public_key`) 从日志中去除。

[NOTE]
====
* 严重性取决于您的日志记录策略：
** *本地日志记录（默认）* — 限制暴露。
** *外部日志记录* — 严重性增加，具体取决于外部日志收集器的安全控制。
* 最终影响的严重性取决于泄露凭据的权限。
====

有关更多信息，请参见 https://attack.mitre.org/techniques/T1552/[不安全的凭据（MITRE ATT&CK T1552）]。

补丁版本包括 `5.4.5` 及以后的版本。升级后旋转用于远程储存库配置的 GitHub 令牌。
